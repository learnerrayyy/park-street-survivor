
# Task1
Due to the current structure of our demo, we set one main task for the participant: to start from the main menu, enter the Room phase, select the items they need, and successfully transition into the DayRun gameplay. The purpose of this task was to evaluate whether new players could understand the early game flow without instructions. Completing this task typically takes around 3–4 minutes.

## Participant Instructions
Starting from the opening screen, please enter the game and complete what you believe is the first full gameplay flow.
- You are free to explore, but try to:  
	- Start the game  
	- Understand what actions are available to you  
	- Leave the room and enter the main running gameplay

## Summary
### Current issues

Players were able to understand the main menu and setting page quickly. The Start button was clear and did not cause confusion. However, once entering the Room phase, hesitation increased. Players were unsure what objects were interactive and spent time walking around without a clear goal. This suggests that the Room lacks strong visual guidance.  
1. When approaching the desk, some players did not immediately recognise it as an interaction point. The prompt icon in the bottom-right corner was too small and easy to overlook.  
2. the backpack interface, players were confused about item selection rules. The “only one buff-item” constraint was not clearly communicated. Some attempted to select multiple items and only realised the restriction after failing.  
3. Another issue occurred when transitioning from the Room to the DayRun phase. Players did not immediately know how to return to the Room from other pages or where the exit door was located. The door marker was not visually distinct enough, which caused brief pauses and uncertainty.

### Improvement methods

Overall, the main problem is not the core gameplay itself, but the clarity of guidance in the early phase. The game currently relies too much on players exploring and inferring rules on their own. We need to:  
1. Strengthen prompts.  
2. Improve visual hierarchy.  
3. Make interaction points and constraints more explicit.
